## Key decisions and rationale (v1-level)

1. **Locked outcome formula** — `int(did_complete_tasks) + int(did_advance_goal) ∈ {0, 1, 2}` is hardcoded across the chain (not user-configurable in v1). Lets downstream LLM rationales quote the formula verbatim without retraining.
2. **Two named case lenses** (`workday` keyed on `case_date`; `goal_pursuit` keyed on `case_goal_id`) — explicit, type-checked entry points instead of free-form case keys.
3. **Frozen dataclasses on every public engine return** — keeps PM4Py/Kuzu/Pydantic types from leaking across module boundaries; makes everything serializable + immutable.
4. **Plain dataclasses + manual validation, no Pydantic** — keeps the dependency surface narrow. Anthropic SDK transitively pulls Pydantic v2 but we don't depend on it explicitly.
5. **Single-writer daemon scaffold** — `tm/daemon.py` enforces the write path via Unix-socket for any future multi-writer scenario (Telegram bot, etc.). v1 agents bypass the daemon for simplicity (see ARCH-2 in Risks).
6. **Per-call sqlite3 connections in repositories** — avoids long-lived connection state; lets tests pass `tmp_path` DBs without fixture-managed Stores.
7. **Migration runner with `-- !pre-txn:` / `-- !post-txn:` pragma directives** (T-FND-07) — supports table rebuilds (e.g. CHECK constraints, FK additions) that need `PRAGMA foreign_keys=OFF` outside the transaction.
8. **Migration exception hierarchy:** `MigrationError` → `MigrationPragmaError` / `MigrationPostTxnError` / `MigrationBodyError` / `MigrationIntegrityError` (T-FND-08, T-FND-09). `except MigrationError:` catches every runner failure mode.
9. **Vocabulary case-symmetry** — `add_alias` lowercases + strips inputs; `resolve` symmetrically lowercases lookups (T-VOC-01-polish closed the case-asymmetry bug surfaced by T-VOC-04).
10. **PrPM guardrails** (T-INT-02) — three independent guards (Objective: strict `>0`; Counterfactual: `≥0.3`; Conformance: `≥0.4`) run unconditionally with no short-circuit so rationale captures every reason.
11. **Single-summary-per-case_date contract** (T-INT-01) — DebriefAgent enforces at most one summary event with `activity='debrief_summary'` per case_date. Required for OutcomeAggregator's per-attribute MAX semantics to compute correctly.
12. **ProcessMiner filters `debrief_summary` by default** (T-INT-01b) — opt-in via `include_summary_events=True`. Prevents the synthetic summary from polluting variant analysis.

## Critical Caveats — must close before first user

1. **`tm init` is a stub.** First-run bootstrap requires manual `SQLiteStore(db_path).apply_pending_migrations()` + `VocabularyRepository.seed_starter_vocabulary()` + `seed_starter_aliases()`. Without these, users hit `no such table: events`. Either implement `tm init` (T-INT-04-init follow-up) or document the manual bootstrap in README. **Estimated effort:** 30 minutes.
2. **README is two lines** (`Status: pre-alpha — see docs/designs/`). No quickstart, no env-var reference (`TM_DB`, `TM_LLM_API_KEY`, `TM_LLM_MONTHLY_CAP_USD`, `TM_MAX_PROACTIVE_SUGGESTIONS_PER_DAY`), no command index. A first user has nothing to follow. (T-INT-04-readme follow-up). **Estimated effort:** 1 hour.
3. **AnthropicAdapter.extract drops the SDK's usage block.** `extract` returns a plain dict; the cost-meter falls back to `output_tokens=0`. Cost ledger systematically under-reports output spend, undermining `CostMeter.check_budget` reliability under live LLM traffic. The bug was hidden in tests because `unittest.mock.Mock` accepts attribute assignment that plain dicts do not. (T-INT-04-llm-extract-usage follow-up). **Estimated effort:** 2 hours — change `LLMClient.extract` Protocol to return a typed object with usage, or wrap the dict in a class that exposes `input_tokens`/`output_tokens`.

## Architectural risks (acknowledged; tracked)

| ID | Severity | Risk |
|---|---|---|
| ARCH-1 | high | Cost ledger under-reports output tokens (Critical Caveat #3) |
| ARCH-2 | medium | Single-writer daemon enforced by convention only — agents bypass via direct repo injection. Fine for v1 single-user CLI; revisit when daemon becomes mandatory |
| ARCH-3 | medium | `events.advances_goal` soft-FK has TOCTOU between SELECT and INSERT. Closes when events table rebuild lands the formal `REFERENCES` (T-PM-02 follow-up) |
| ARCH-4 | medium | OutcomeAggregator MAX-across-events for `planned_tasks_completed/total` could mix attributes from different summary events. Contract (one summary event per case_date) is implicit; enforced in DebriefAgent but not asserted at write time |
| ARCH-5 | low | Magic strings shared by reference across modules (`workday`/`goal_pursuit` lens names, cluster labels). Renames require grep-wide sweeps. Labels are LLM-quotable per the locked-formula decision |
| ARCH-6 | low | Three repository write paths overlap (tm/store.py, tm/stores/sqlite_store.py, tm/repositories/events.py). Acknowledged |
| ARCH-7 | low | Kuzu projection borrows `ProcessMiner._load_dataframe` (private) with `# noqa: SLF001` |
| ARCH-8 | low | `EventsRepository.query_events` `until` is exclusive but `list_distinct_case_dates` `until` is inclusive — asymmetric footgun. Both behaviors are documented; module-level invariant note recommended |
